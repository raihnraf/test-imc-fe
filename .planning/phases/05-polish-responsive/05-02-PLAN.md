<objective>
Make CRUD tables and forms responsive across desktop and tablet screen sizes. Tables dynamically hide secondary columns on tablet using computed `displayedColumns` signals. Forms extend their single-column media query breakpoint from 600px to 959px to cover the tablet range.

Purpose: Tables with 5-6 columns overflow on tablet screens. Forms with two-column grid are cramped on tablet. Both need responsive adaptation for the admin dashboard to be usable on tablet-width viewports.
Output: Responsive tables with dynamic column hiding, responsive forms with extended breakpoint, table overflow wrappers, and global responsive utilities.
</objective>

    1. Add imports: `BreakpointObserver, Breakpoints` from `@angular/cdk/layout`; `toSignal` from `@angular/core/rxjs-interop`; `map, startWith` from `rxjs`. Also add `computed` to the existing `@angular/core` import if not already present.

    In each of the 3 form SCSS files (user-form, level-form, page-form):

    1. Change the existing `@media (max-width: 600px)` breakpoint to `@media (max-width: 959px)`. This extends the single-column form layout from phone-only to cover the full tablet range (Small + TabletPortrait per Material breakpoints).

       Before:
       ```scss
       @media (max-width: 600px) {
         .user-form { grid-template-columns: 1fr; }
       }
       ```

       After:
       ```scss
       @media (max-width: 959px) {
         .user-form { grid-template-columns: 1fr; }
       }
       ```

       Apply the same change to level-form.component.scss (targeting .level-form) and page-form.component.scss (targeting .page-form).

    In styles.scss:

    2. Add global responsive utility rules after the existing html/body reset:
       ```scss
       img {
         max-width: 100%;
         height: auto;
       }

       .table-wrapper {
         overflow-x: auto;
         -webkit-overflow-scrolling: touch;
       }
       ```

       The `.table-wrapper` class is a shared utility that any component can use for horizontal table scroll fallback. The `img` rule prevents images from overflowing their containers on small screens.
