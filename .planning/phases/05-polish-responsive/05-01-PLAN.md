<objective>
Make the admin layout (sidenav, toolbar, content area) responsive across desktop and tablet screen sizes using BreakpointObserver from @angular/cdk/layout. The sidenav switches from `mode="side"` (desktop) to `mode="over"` (tablet) with a hamburger toggle button in the toolbar.

Purpose: This is the primary gap identified in research — the sidenav is currently hardcoded to `mode="side"` with fixed 250px width, causing horizontal scrolling or cramped content on tablet screens.
Output: Responsive admin layout with BreakpointObserver-driven sidenav mode switching, hamburger toggle, and tablet-optimized content padding.
</objective>

    In admin-layout.component.ts:

    1. Add imports: `BreakpointObserver, Breakpoints` from `@angular/cdk/layout`; `toSignal` from `@angular/core/rxjs-interop`; `map, startWith` from `rxjs`.

    2. Add private field: `private readonly breakpointObserver = inject(BreakpointObserver)`.

    3. Replace the existing `readonly sidenavOpen = signal(true)` with three new signals:
       - `readonly isTablet = toSignal(this.breakpointObserver.observe([Breakpoints.Handset, Breakpoints.TabletPortrait]).pipe(map(result => result.matches), startWith(false)), { initialValue: false })` — uses toSignal to convert BreakpointObserver observable to a signal, no manual subscription needed (avoids Pitfall 2: memory leaks per RESEARCH.md).
       - `readonly sidenavMode = computed(() => this.isTablet() ? 'over' : 'side')` — derived sidenav mode.
       - `readonly sidenavOpened = signal(true)` — mutable signal for open/close state, initialized to true (desktop default).

    4. Add `toggleSidenav()` method: `this.sidenavOpened.update(v => !v)`.

    5. Remove the unused `sidenavOpen` signal (replaced by `sidenavOpened`).

    In admin-layout.component.html:

    6. Replace `[mode]="'side'"` with `[mode]="sidenavMode()"` on the mat-sidenav element.

    7. Replace `[opened]="sidenavOpen()"` with `[opened]="sidenavOpened()"` on the mat-sidenav element.

    8. Add a hamburger button inside mat-toolbar, before the `.toolbar-title` span:
       - Use `mat-icon-button` with `(click)="toggleSidenav()"`
       - Wrap in `@if (isTablet()) { ... }` control flow so it only renders on tablet
       - Contains `<mat-icon>menu</mat-icon>`
       - Add `aria-label="Toggle navigation"` for accessibility

    Do NOT add `hasBackdrop` to mat-sidenav-container — let mode="over" use its default backdrop behavior (avoids Pitfall 1 per RESEARCH.md). Do NOT use `mode="push"` — use "over" for tablet (avoids anti-pattern per RESEARCH.md).
  </action>
