<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Login form → AuthService | User credentials enter form; sent to backend over HTTP (recommend HTTPS in production) |
| APP_INITIALIZER → Session restoration | App blocks until session check completes; failed refresh clears sessionStorage silently |
| Router → Auth guard | Unauthenticated navigation attempts intercepted and redirected |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-09 | Information Disclosure | login.component.ts password field | mitigate | Use `type="password"` on MatInput for password field; Angular's built-in masking prevents shoulder-surfing |
| T-01-10 | Spoofing | APP_INITIALIZER session restoration | mitigate | restoreSession() validates refresh token with backend; if token is expired/revoked, sessionStorage is cleared and user starts fresh |
| T-01-11 | Tampering | app.routes.ts route configuration | accept | Client-side route config is UX only; backend enforces permissions on every API call. Unauthorized direct URL access blocked by authGuard + backend checks. |
| T-01-SC | Tampering | npm package installs | mitigate | All packages verified in RESEARCH.md Package Legitimacy Audit (all Approved, official Angular ecosystem) |
</threat_model>
